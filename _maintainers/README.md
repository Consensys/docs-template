# Maintainers Directory

This directory contains repository-specific configurations, overrides, and metadata for content porting.

## Structure

```
_maintainers/
├── link-replacements.yaml  # Git-tracked: Link rewriting rules
└── logs/                    # Git-ignored: Build outputs, sync logs, debug info
    ├── component-import-fixes.log
    ├── links-dropped.log
    ├── transformation-summary.log
    └── last-run-timing.json
```

## Git Tracking

- **`_maintainers/` (git-tracked, shared)**: Repo-specific configs, overrides, partial metadata. PRs include these changes explicitly—reviewers see "this repo needed X tweak for CreditCost".

- **`logs/` (git-ignored, local)**: Build outputs, sync logs, debug info. Generated by `npm run port` → `logs/sync-*.json`. Regenerate on clone.

## Configuration Files

### `link-replacements.yaml`

YAML configuration for link rewriting rules. Defines:
- Exact link replacements
- Pattern-based replacements with network placeholders
- External link preservation rules

See `docs/single-source/between-repos/Plugins/port-data.md` for configuration details.

## Log Files

All log files in `logs/` are git-ignored and regenerated on each build:

- `component-import-fixes.log` - Component import transformations
- `links-dropped.log` - Links that were removed (alphabetically sorted)
- `transformation-summary.log` - Summary of all transformations
- `last-run-timing.json` - Execution time tracking
